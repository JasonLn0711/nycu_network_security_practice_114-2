# Handout Note: Introduction

## Scope

This handout is a course-orientation slide deck. Its job is not deep technical coverage; it tells you what kind of security thinking the course expects, how the course is graded, and how the topics are staged across the semester.

## Key Points

- The course is positioned as a doorway into attack-and-defense research, not just a list of isolated attacks.
- The topic spread is broad: system security, network security, software security, web security, hardware security, AI security, and cryptography all appear in the roadmap.
- The prerequisites matter: operating systems, computer networks, and coding ability are part of the expected baseline.
- Course evaluation is homework-heavy, so steady weekly study will matter more than last-minute review.

## Grading And Workflow

- Homework: 50%
- Midterm: 20%
- Term project: 30%
- Homework is individual work.
- The term project is group work.
- E3 is used for materials and grading.

Study implication:

- Build technical foundations early, especially OS and networking.
- Keep notes by topic, because later attack and defense chapters reuse the same system concepts.
- Treat the handouts as the canonical topic order and your Markdown files as the place for explanation.

## Topic Map

The handout divides the course into several layers:

- System-level attacks: malware, environment variables
- System-level defenses: Linux security basics, Set-UID, access control models, Bell-LaPadula, SELinux, Windows security
- Network-level attacks: UDP, TCP, MAC/ARP, DNS security
- Network-level defenses: IDS, firewalls, crypto primitives, hash/MAC, TLS
- Software-level attacks: buffer overflow, return-to-libc, format string
- Web-level attacks: CSRF, XSS, SQL injection

## What To Remember

- This course is cumulative. Early system concepts become prerequisites for later exploit and defense topics.
- "Attack and defense" here means understanding how mechanisms work, how they fail, and how designs can be hardened.
- If a later handout feels difficult, come back to the prerequisites listed here rather than memorizing the later slides blindly.

## Review Checklist

- Can I explain why OS, networking, and programming are prerequisite knowledge?
- Do I know which topics belong to system, network, software, and web security?
- Do I have a note-taking structure that matches the course topic map?
