# Week 01 Key Concepts

## Course Frame

Network security practice studies how systems fail, how attackers turn failures into leverage, and how defenders reduce risk with controls, monitoring, and evidence.

## Core Concepts

- Operating system: manages processes, memory, files, devices, and privilege.
- Kernel: privileged core that mediates system resources.
- System call: controlled entry point from user programs into the kernel.
- Process: a program in execution.
- User mode and kernel mode: the privilege boundary that limits ordinary programs.
- Evidence: commands, logs, packets, screenshots, or written observations that support a security conclusion.

## Exam Cue

When asked about an attack, name the abused mechanism first. Then explain the privilege, data, or network boundary it crosses.
