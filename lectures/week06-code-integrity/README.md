# Week 06: Code Integrity

This week studies how systems decide whether code should be trusted, loaded, executed, or blocked.

## Start Here

1. [key-concepts.md](key-concepts.md)
2. [examples.md](examples.md)

## Students Should Be Able To

- Explain why code identity is different from file identity.
- Connect hashes, signatures, loaders, policies, and update channels.
- Describe how integrity controls fail when trust roots or signing workflows are weak.
