# Sentinel Documentation

This folder holds durable design and requirement notes. Keep runnable demo steps
in `../demo/`, generated evidence in `../reports/`, and final submission files
in `../report/`.

Language-specific notes are split by implementation path: `python/` documents
the primary release-gated scanner, while `rust/` documents the optional systems
language companion.

| File | Use |
| --- | --- |
| `requirements-traceability.md` | Maps the official brief to implementation and evidence. |
| `python/technical-design.md` | Explains the primary Python scanner architecture, data structures, and release checks. |
| `rust/README.md` | Documents the optional Rust companion implementation under `../rust/`. |
| `standards-alignment.md` | Records the EICAR, NIST, OWASP, and MITRE references used to calibrate safety and claims. |
