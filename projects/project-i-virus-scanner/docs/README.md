# Sentinel Documentation

This folder holds durable design and requirement notes. Keep runnable demo steps
in `../demo/`, generated evidence in `../reports/`, and final submission files
in `../report/`.

The current submission path is the Rust implementation documented in `rust/`.
The Project I evidence points to the Rust scanner, Rust reports, Rust demo
runbook, and Rust verification commands.

Git history preserves the removed Python prototype. Do not reintroduce a second
live implementation unless the instructor requires it.

| File | Use |
| --- | --- |
| `requirements-traceability.md` | Maps the official brief to implementation and evidence. |
| `rust/README.md` | Documents the Rust scanner implementation under `../rust/`. |
| `standards-alignment.md` | Records the EICAR, NIST, OWASP, and MITRE references used to calibrate safety and claims. |
