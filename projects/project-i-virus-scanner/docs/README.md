# Sentinel Documentation

This folder holds durable design and requirement notes. Keep runnable demo steps
in `../demo/`, generated evidence in `../reports/`, and final submission files
in `../report/`.

| File | Use |
| --- | --- |
| `requirements-traceability.md` | Maps the official brief to implementation and evidence. |
| `technical-design.md` | Explains scanner architecture, data structures, and release checks. |
| `standards-alignment.md` | Records the EICAR, NIST, OWASP, and MITRE references used to calibrate safety and claims. |
