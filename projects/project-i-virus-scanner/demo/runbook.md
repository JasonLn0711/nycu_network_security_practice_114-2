# Demo Runbook

## Purpose

Show that the Rust Sentinel scanner can scan a small directory tree, generate
the official EICAR safe anti-malware test file for the controlled demo, detect
that EICAR file, keep clean files clean, separate heuristic warnings from
confirmed signatures, and produce reproducible JSON/Markdown evidence.

Current Rust checkpoint:

- `cargo fmt --check` passes.
- `cargo test` passes with `13` tests.
- `cargo clippy --all-targets -- -D warnings` passes.
- `cargo run -- prepare-eicar-demo ...` writes the exact 68-byte EICAR file
  into the nested demo tree.
- `cargo run -- verify-demo ...` passes with the expected Project I summary.
- `cargo run -- write-evidence ...` writes the Rust reproducibility manifest.

## Safety Rules

- Use only the official EICAR safe anti-malware test file and benign local
  fixtures.
- Do not download, store, execute, or analyze live malware.
- Do not scan personal directories or third-party files.
- Do not delete, quarantine, upload, or modify scanned files.
- Keep symbolic links skipped by default.
- Generate the EICAR fixture with Rust at demo time; do not commit or export the
  generated `eicar.com.txt` file.

## Demo Tree

```text
demo/demo-tree/
  clean/
    notes.txt
    image-placeholder.bin
  nested/
    level-1/
      level-2/
        eicar.com.txt   # generated by cargo run -- prepare-eicar-demo
  suspicious/
    api-names-fixture.txt
  ignored-or-empty/
    empty.txt
```

| Path | Expected status | Reason |
| --- | --- | --- |
| `clean/notes.txt` | clean | Benign text fixture. |
| `clean/image-placeholder.bin` | clean | Binary-like fixture without a signature. |
| `nested/level-1/level-2/eicar.com.txt` | infected | Official EICAR safe test file; MD5, SHA-256, and hex-pattern match. |
| `suspicious/api-names-fixture.txt` | suspicious | Heuristic-only API-name signal. |
| `ignored-or-empty/empty.txt` | clean | Empty file should not break scanning. |

## Run Sequence

1. State the safety boundary: read-only scanner, generated EICAR safe test file
   only, no live malware.
2. Show `signatures/malware-signatures.json`.
3. Show `rust/src/main.rs`, `rust/src/scanner.rs`, `rust/src/signatures.rs`,
   `rust/src/patterns.rs`, and `rust/src/reporting.rs`.
4. Show the demo tree.
5. Run the Rust EICAR preparation command or `make rust-prepare-eicar`.
6. Run the Rust verification gate.
7. Run the Rust scan command to regenerate `reports/demo-report.json` and
   `reports/demo-report.md`.
8. Run the Rust evidence command to regenerate
   `reports/demo-evidence-manifest.json`.
9. Open `reports/demo-report.md`.
10. Point out the infected EICAR fixture, suspicious fixture, and clean files.
11. Point out scan metadata: Bloom-filter hash pre-check, exact hash-map
   verification, Aho-Corasick pattern engine, heuristic rules, and
   `symlink_policy: skip`.
12. State the final demo decision: the demo satisfies the EICAR requirement by
    generating and scanning the official EICAR safe test file.
13. State the private-repo decision: the implementation was not moved to a
    private remote in this pass, by instruction; the curated export package is
    ready.
14. Close with the limitation: educational signature scanner, not production
    antivirus.

## Rust Commands

Run from `projects/project-i-virus-scanner/rust/`.

```bash
cargo fmt --check
cargo test
cargo clippy --all-targets -- -D warnings
cargo run -- validate-signatures --signatures ../signatures/malware-signatures.json
cargo run -- prepare-eicar-demo --target ../demo/demo-tree
cargo run -- verify-demo \
  --target ../demo/demo-tree \
  --signatures ../signatures/malware-signatures.json
cargo run -- scan \
  --target ../demo/demo-tree \
  --signatures ../signatures/malware-signatures.json \
  --json ../reports/demo-report.json \
  --markdown ../reports/demo-report.md
cargo run -- write-evidence \
  --target ../demo/demo-tree \
  --signatures ../signatures/malware-signatures.json \
  --report ../reports/demo-report.json \
  --report ../reports/demo-report.md \
  --output ../reports/demo-evidence-manifest.json
```

Expected verification summary:

```text
Rust demo verification passed: scanned=5 infected=1 suspicious=1 clean=3 skipped=0 errors=0
```

Expected scan summary:

```text
Sentinel v2 Rust scan complete: scanned=5 infected=1 suspicious=1 clean=3 errors=0
```

The scan command returns exit code `1` because the generated EICAR safe test
file is intentionally detected.
